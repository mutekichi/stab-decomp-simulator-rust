# Contributing

Thank you for your interest in contributing to NECSTAR!
Contributions of all kinds are welcome and appreciated.

## How to contribute

You may contribute in any of the following ways:

- [Open an issue](https://github.com/mutekichi/necstar/issues) on GitHub
  to report a bug or propose an enhancement
- [Submit a pull request](https://github.com/mutekichi/necstar/pulls) with
  your proposed changes
- Contact the maintainer directly at
  [ywatanabe.r5p@gmail.com](mailto:ywatanabe.r5p@gmail.com)

Before opening a new issue or pull request, please check
[existing issues](https://github.com/mutekichi/necstar/issues)
and [pull requests](https://github.com/mutekichi/necstar/pulls)
to avoid duplication.

## Development guidelines

When submitting a pull request, the following items are not strictly
required, but addressing them may significantly increase the chance
of a smooth and timely review and approval:

- Code formatting consistent with the existing style
- Addition or update of tests relevant to the proposed changes
- Addition or update of examples, when applicable
- For changes affecting the Rust API, corresponding updates to
  Python bindings, including appropriate Python type annotations
  or stub files, when relevant

Contributors are encouraged to include these improvements when they
are reasonably within scope of the proposed changes.

Please note that during the review process, maintainers may make
additional changes to a pull request, including but not limited to:

- Formatting or refactoring
- Adding or adjusting tests
- Updating examples or documentation
- Minor API or binding-related fixes

These changes are intended to improve code quality, consistency,
and long-term maintainability. By submitting a pull request, you
acknowledge and agree to this review process.

## Code of conduct

Please keep all discussions respectful and constructive.
